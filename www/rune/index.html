rune

Beta

Automatic differentiation and JIT compilation with composable transformations, inspired by JAX.

Get Started View Examples

Core Transformations

grad

Automatic differentiation for computing gradients of scalar-valued functions.

let f x = Tensor.sum (Tensor.square x)
let grad_f = grad f

jit

Just-in-time compilation for optimized execution on various backends.

let fast_f = jit f
let result = fast_f input

Composition

Transformations compose seamlessly for complex optimization workflows.

let optimized = grad (jit f)
let parallel = vmap (jit f)

Advanced Features

Reverse-Mode AD

Efficient backpropagation for neural networks and optimization with O(1) gradient computation.

Multi-Device

Seamless execution across CPU, GPU (Metal/CUDA), and accelerators with automatic memory management.

Vectorization

vmap for automatic batching and parallel execution over leading dimensions.

Higher-Order Derivatives

Compose grad transformations for Hessians and higher-order optimization methods.

Type Safety

OCaml's type system ensures gradient shapes and device placement correctness at compile time.

Functional API

Pure functional transformations with immutable tensors and composable operations.

Neural Network Training

(* Define a simple MLP *)
open Rune

let mlp params x =
  x
  |> Tensor.linear params.w1 params.b1
  |> Tensor.relu
  |> Tensor.linear params.w2 params.b2

(* Loss function *)
let loss params (x, y) =
  let pred = mlp params x in
  Tensor.mse_loss pred y

(* Training step with automatic differentiation *)
let step params batch =
  let grad_fn = grad loss in
  let grads = grad_fn params batch in
  Optimizer.sgd params grads ~lr:0.01

(* JIT compile for performance *)
let fast_step = jit step

Available Transformations

Differentiation

  • grad - Reverse-mode automatic differentiation
  • jacrev - Jacobian via reverse-mode AD
  • jacfwd - Jacobian via forward-mode AD
  • hessian - Second-order derivatives

Compilation

  • jit - Just-in-time compilation
  • aot - Ahead-of-time compilation
  • device - Device placement control
  • precision - Numerical precision control

Vectorization

  • vmap - Vectorize over batch dimension
  • pmap - Parallel map across devices
  • scan - Sequential scan operations
  • while_loop - Optimized loops

Debugging

  • checkify - Numerical error checking
  • debug - Debug mode with tracing
  • profile - Performance profiling
  • visualize - Computation graph visualization

Rune Ecosystem

High-level libraries built on Rune's automatic differentiation and JIT compilation for specialized machine learning tasks.

Kaun

Beta

Neural Network Library

High-level neural network library with pre-built layers, optimizers, and training loops. The Flax equivalent for the Rune ecosystem.

Key Features:

  • • Modular Architecture: Composable layers and models
  • • Built-in Optimizers: SGD, Adam, RMSprop with learning rate schedules
  • • Training Utilities: Automated training loops and checkpointing
  • • Model Zoo: Pre-trained models for common tasks
  • • Transfer Learning: Easy fine-tuning and adaptation
  • • Distributed Training: Multi-GPU and multi-node support
(* Define a simple MLP with Kaun *)
open Kaun
let model = Sequential.[
Linear.create ~in_features:784 ~out_features:128;
Activation.relu;
Linear.create ~in_features:128 ~out_features:10;
]
let trainer = Trainer.create ~model ~optimizer:Adam.default

Sowilo

Alpha

Computer Vision Library

Differentiable computer vision operations with JIT compilation and GPU acceleration for real-time processing.

Key Features:

  • • Differentiable Operations: All CV ops support automatic differentiation
  • • JIT Compilation: Optimized kernels for GPU acceleration
  • • Real-time Processing: Efficient video and camera input handling
  • • Pre-trained Models: Object detection, segmentation, pose estimation
  • • Custom Pipelines: Composable image processing workflows
  • • Multi-device: CPU, Metal (macOS), CUDA (Linux) backends
(* Differentiable image processing *)
open Sowilo
let pipeline img =
img
|> gaussian_blur ~sigma:1.0
|> edge_detection ~method:Canny
|> morphology ~op:Closing
let jit_pipeline = jit pipeline

Design Philosophy

The Rune ecosystem follows the same philosophy as JAX/Flax: provide powerful low-level primitives (grad, jit, vmap) and build specialized high-level libraries on top.

Transformations

All ecosystem libraries benefit from Rune's grad, jit, and vmap transformations.

Composability

Libraries interoperate seamlessly, sharing tensors and computation graphs.

Type Safety

OCaml's type system ensures correctness across the entire ecosystem.
