# Input and output

`Nx_io` loads and saves tensors in image, NumPy, SafeTensors, and delimited text
formats. The examples below focus on images and NumPy files.

## Images

`load_image` recognizes static PNG and JPEG streams from their contents. It
returns an RGB `uint8` tensor by default, or a two-dimensional grayscale tensor
when requested.

<!-- $MDX skip -->
```ocaml
let rgb = Nx_io.load_image "photo.png"
(* Nx.shape rgb = [| height; width; 3 |] *)

let gray = Nx_io.load_image ~grayscale:true "photo.jpg"
(* Nx.shape gray = [| height; width |] *)
```

`save_image` selects PNG or JPEG from the destination extension. It accepts
`uint8` tensors with these shapes:

- `[| height; width |]` or `[| height; width; 1 |]` for grayscale;
- `[| height; width; 3 |]` for RGB;
- `[| height; width; 4 |]` for PNG RGBA output.

<!-- $MDX skip -->
```ocaml
Nx_io.save_image "copy.png" rgb;
Nx_io.save_image ~overwrite:false "thumbnail.jpg" thumbnail
```

The default is to replace an existing destination atomically. Pass
`~overwrite:false` when replacing an existing file would be an error.

## NumPy arrays

### NPY files

An NPY file carries a dtype that is known only at runtime, so `load_npy` returns
an existentially packed tensor.

<!-- $MDX skip -->
```ocaml
let packed = Nx_io.load_npy "data.npy"
let shape = Nx_io.packed_shape packed

let data = Nx_io.to_typed Nx.float32 packed
(* [to_typed] raises [Failure] if the file contains another dtype. *)

Nx_io.save_npy "copy.npy" data
```

Pattern matching is useful when code can operate without fixing the dtype:

<!-- $MDX skip -->
```ocaml
let describe path =
  let Nx_io.P tensor = Nx_io.load_npy path in
  (Nx.dtype tensor, Nx.shape tensor)
```

NPY versions 1.0, 2.0, and 3.0 are accepted. Nx reads and writes boolean,
16/32/64-bit floating-point, 8/16/32/64-bit signed and unsigned integer, and
64/128-bit complex arrays. Both C and Fortran input order and either byte order
are accepted. Dtypes without a standard NPY representation, such as bfloat16,
float8, and 4-bit integers, cannot be saved.

### NPZ archives

An NPZ archive maps names to packed tensors. `load_npz` loads all NPY entries;
`load_npz_entry` avoids decoding entries that are not needed.

<!-- $MDX skip -->
```ocaml
let archive = Nx_io.load_npz "model.npz" in
let weights =
  Hashtbl.find archive "weights" |> Nx_io.to_typed Nx.float32
in

let labels =
  Nx_io.load_npz_entry ~name:"labels" "model.npz"
  |> Nx_io.to_typed Nx.int64
in

Nx_io.save_npz "checkpoint.npz"
  [ ("weights", Nx_io.P weights); ("labels", Nx_io.P labels) ]
```

Names passed to `save_npz` are logical names without the `.npy` suffix. They
must be unique, valid UTF-8, and safe relative paths. Nx selects Store or
DEFLATE compression independently for each entry.

## Gzip streams

`gunzip` validates all members and their checksums before replacing the
destination. Concatenated members are supported.

<!-- $MDX skip -->
```ocaml
Nx_io.gunzip ~src:"dataset.gz" ~dst:"dataset.bin"
```

## Errors and replacement

Malformed formats, unsupported dtypes, invalid archive names, and dtype
mismatches raise `Failure`. NumPy, SafeTensors, and text helpers also report
filesystem failures as `Failure`; image and gzip operations preserve
`Unix.Unix_error` for filesystem failures.

`save_npy`, `save_npz`, and `save_image` replace an existing destination by
default. Their `~overwrite:false` mode requires the destination not to exist.
Writes that replace a destination are completed through a temporary sibling so
an encoding failure does not leave a partially replaced file.

## Implementation and performance

Nx owns the ISC image, DEFLATE, and ZIP implementations used by `nx.io`; it does
not require system zlib, libpng, libjpeg, or pkg-config. Image and archive
decoding writes into the final Nx buffer. Use `load_npz_entry` for large
archives when only one tensor is needed.
